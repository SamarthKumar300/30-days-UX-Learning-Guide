# V2 Guidebook Plan: Complete Website Content Master Plan

> **Target:** Junior Product Designer (1 year experience) -> 25 LPA+ product-based companies
> **Duration:** 8 weeks / 56 days
> **Primary delivery surface:** The website itself
> **Content inputs:** Existing live website content + complete PDF material + `content/Day-wise-content.md` conversion style
> **Core rule:** The learner should study from the website, not from the PDFs

---

## 1. Purpose

This document is the canonical blueprint for how the V2 website content must be structured, merged, and delivered.

It is **not** a PDF sourcing map.
It is **not** a reminder to link users to source files.
It is **not** a plan where learner-facing study depends on opening PDFs from a references page.

The final website must contain the actual teaching material inside the day pages.

That material must combine:

- the strongest insights already live on the website
- the full set of meaningful ideas from all five PDFs
- the clearer, web-native teaching style already demonstrated in `content/Day-wise-content.md`

If the same topic appears in both the live website and the PDFs, it must be merged into one stronger teaching flow.

If a point appears only in one source, it must be preserved.

No meaningful concept, framework, example, drill, challenge, scenario, or interview-useful idea from the PDFs should be dropped.

---

## 2. Non-Negotiable Product Rules

### Website-first learning

- The website is the primary study experience.
- The learner should be able to complete the program end-to-end without needing to open PDFs.
- PDFs are source assets for content creation, not required study steps.

### Main study block is the canonical teaching surface

For every day page:

- the main study block must contain the actual instructional content
- it must not be a short teaser, summary stub, or sourcing label
- it must not depend on an "extended guide" below to become useful

If supporting blocks exist, they support the study block. They do not replace it.

### Existing live website content is mandatory input

- Existing website insights are not filler and are not to be overwritten casually.
- Every current live insight must be reviewed and either:
  - preserved directly
  - merged into a stronger rewritten section
  - absorbed into a clearer explanation without losing the substance

### Full PDF coverage is mandatory

- Every meaningful point from the PDFs must be represented somewhere in the website curriculum.
- Overlap is solved by merging, not by omission.
- No chapter may be reduced to a label-only placeholder in the final-state plan.

## 4. Merge Rules

Apply these rules throughout the entire 56-day curriculum.

### Topic merge rule

If the live website and PDF content cover the same concept:

- create one merged section
- use the sharper explanation as the backbone
- fold in missing nuance, examples, frameworks, and interview angles from the other source
- remove repeated sentences and parallel sections

### Repetition rule

- Avoid accidental repetition across days.
- Keep repetition only when it is deliberate reinforcement with a different lens, such as:
  - concept -> application
  - framework -> interview use
  - theory -> critique practice
  - individual skill -> integrated simulation

### Strength rule

- If the live website explains a topic better than the PDF, keep the live version and absorb any missing PDF nuance.
- If the PDF explains a topic better, use the PDF-derived explanation and absorb any unique website insight.
- If both are strong, merge both.

### Coverage rule

- Never delete a meaningful point only because it feels similar to something nearby.
- Similar points should be consolidated carefully so no distinction is lost.

## 5. Content Contract for Every Day

Each of the 56 days must be authored as a complete website-native learning unit.

### Required day structure

Each day page must contain:

1. **Main study block**
   - the actual teaching content for that day
   - enough depth that the learner can study on-site without opening external files
   - merged content from live website + PDFs where relevant

2. **Practice block**
   - drills, frameworks, or scenario-based application

3. **Assignment block**
   - concrete work to complete that day

4. **Interview readiness layer**
   - mindset
   - answer patterns
   - proof exercise

5. **Optional supporting systems**
   - challenge cards
   - trap cards
   - defense drills
   - behavioral scenarios
   - mock interview components

### Study block standard

The main study block should generally be written in the same style demonstrated by `content/Day-wise-content.md`:

- teaching-first prose
- explicit conceptual explanation
- concrete frameworks
- practical examples
- interview angles where relevant

It may be rendered as one continuous study block or internally segmented, but the learner must feel they are reading the actual curriculum, not metadata about the curriculum.

### References rule

- `references.html` may exist only as optional support.
- It must not function as "read these PDFs to study."
- PDFs, if still exposed anywhere, are optional source assets and not part of the normal learner flow.

## 11. Technical Content Delivery Directives

These directives exist to remove ambiguity about final behavior.

### Main study block

- The main study block on each day page is the canonical curriculum surface.
- PDF-derived content should replace or become the primary body of that block.
- The site should not rely on an appended "extended study guide" to make the day useful.

### Extended study content

- If an "extended study guide" treatment remains temporarily during migration, it is transitional only.
- Final-state UX should present the day's real content as the main study experience.

### References page

- `references.html` may remain only as optional support.
- It must not be framed as the place where users go to study the core curriculum.
- If PDF links remain visible, they must be clearly secondary and not required to progress.

### Content data architecture

Future content files should follow one rule:

- the data feeding the day pages must contain full website-readable instructional material
- not just tags, labels, or external-source pointers

## 13. Final Directive

When there is a choice between:

- preserving a source label
- or delivering real website content

choose real website content.

When there is a choice between:

- keeping duplicate explanations side by side
- or merging them into one stronger section

choose the merged section.

When there is a choice between:

- simplifying the curriculum by dropping a PDF point
- or preserving that point through careful integration

preserve the point.

The final V2 guidebook must feel like one coherent website-native curriculum built from all available material, not a website that tells the learner to leave the website and read source documents.
